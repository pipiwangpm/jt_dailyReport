# 后续实施路线

## Phase 2 设计稿

输入：确认后的 Phase 1 原型。

产出：
- 高保真页面设计。
- 组件规范：按钮、输入框、表格、标签、状态色、弹窗、空状态。
- 关键交互状态：加载、成功、失败、空数据、筛选无结果。
- 智能收件箱状态：解析中、待确认、已入库、低置信度、可撤回。
- 移动端或窄屏适配策略。

设计原则：
- 工作台工具气质，信息清晰、克制、适合长期反复使用。
- 首页不做营销表达，直接进入可操作页面。
- 使用状态色区分待办、进行中、完成、延期、取消。
- 图表服务于判断和复盘，不做装饰性图形。

## Phase 3 前端原型

输入：确认后的设计稿。

建议技术：
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- Recharts

范围：
- 使用本地模拟数据。
- 完成可点击页面和主要交互。
- 支持自然语言录入、候选事项确认、新增事项、筛选事项、切换状态、生成报告、复制报告。
- 不接真实数据库、登录、钉钉或 AI。

验收：
- 页面可本地运行。
- 用户可以完整走通“记录事项 -> 查看看板 -> 生成报告”的流程。
- 视觉、交互和信息密度接近最终产品。

## Phase 4 后端服务

输入：确认后的前端原型。

当前本地版：
- 已提供 `server.js` 本地服务，先用 `data/daily-records.json` 持久化。
- 已支持页面静态服务、记录读取、自然语言解析、确认入库、日报/周报/月报生成。
- 已增加 `data/project-kb.json` 项目知识库、钉钉接收预留接口、钉钉发送接口和人工审查后发送流程。
- 本地访问地址为 `http://127.0.0.1:4173`。

建议技术：
- PostgreSQL
- Prisma
- Docker Compose
- 国内云服务器
- Caddy 或 Nginx

范围：
- 事项、项目、标签、报告模板持久化。
- 当前对话/钉钉机器人输入的原文、解析结果和确认状态持久化。
- 报告历史保存。
- 访问保护。
- 钉钉提醒和钉钉消息接收。
- 可选 AI 润色。

上线前检查：
- 数据库备份策略。
- 环境变量与密钥管理。
- 钉钉 Webhook 测试。
- HTTPS 配置。
- 服务重启后数据保留。
